# Practice

`practice/` — internal/course source для `.cpp` и `.hpp` заготовок задач.

Это не обязательная пользовательская папка сайта. Обычный пользователь должен понимать задачу и видеть стартовый код прямо на странице `/tasks/<task-id>`, без инструкции “найди файл в `practice`”.

## Для чего нужна папка

- хранить исходные заготовки задач рядом с репозиторием;
- связывать task data с реальными `.cpp/.hpp` файлами;
- упростить будущие функции скачивания практики;
- подготовить базу для браузерного редактора, backend/autochecker или sandbox;
- дать разработчику и автору курса проверяемый источник starter code.

## Как использовать сейчас

Для обычного обучения на сайте:

1. Открыть задачу на сайте.
2. Прочитать условие.
3. Использовать стартовый код, показанный на странице задачи.
4. Проверить результат по блоку “Готово, когда” и “Самопроверка”.

Для локальной разработки курса:

1. Добавить или обновить `.cpp/.hpp` заготовку в `practice`.
2. Подключить её в `site/src/data/tasks.ts` как internal `practicePath`/`fileName`.
3. Убедиться, что task page показывает понятный starter code без требования вручную искать путь.
4. Запустить frontend checks.

## Что не делать

- Не удалять `practice`: это часть course source.
- Не показывать `practice/...` обычному пользователю как обязательный шаг.
- Не дублировать списки practice-файлов в theory content, если CoursePage уже показывает системный блок задач.
- Не хранить здесь решения с секретами, артефакты сборки или пользовательские данные.
