# Homepage Landing Plan

Цель главной страницы: быстро объяснить новичку, что C++ можно начать с понятного маршрута, а не с поиска по разделам.

## Current Standard

HomePage для `/` — публичная главная без app-shell.

- Не показывать sidebar, search, рабочий topbar и AI assistant.
- Не делать hero как code editor или dashboard.
- Главный visual — карта первого маршрута: линия, короткие станции и одна active-card.
- Hero даёт один основной старт, остальные входы живут в отдельной секции.

## First Screen

- H1: `C++ становится понятнее, когда есть маршрут`.
- Lead: `Uchicode ведёт от первого шага к задачам: короткое объяснение, простой пример и практика в одном месте.`
- Primary CTA: `Начать с нуля`.
- Secondary link: `Посмотреть задачи`.
- Header: shared `BrandLogo`, `Курсы`, `Задачи`, login/profile, optional theme switcher.

## Route Map

Карта показывает 5 шагов:

1. Старт.
2. Первая тема.
3. Простой пример.
4. Частая ошибка.
5. Первая задача.

Правила:

- одна активная станция;
- лёгкая линия маршрута;
- 1-3 слова пояснения у станции;
- одна active-card с пояснением текущего шага;
- без сложного кода в первом visual;
- без пяти одинаковых тяжёлых карточек;
- без больших тёмных панелей и декоративного шума.

## Page Sections

Максимум 5 основных блоков:

1. Public hero + карта маршрута.
2. Один урок - один шаг.
3. Выбор подходящего входа.
4. Помощь рядом с темой.
5. Compact status: открытые темы и задачи.

Не добавлять нижний CTA, если он повторяет hero.
Кликабельные route cards должны иметь видимую строку действия вроде `Начать базу ->`; информационные карточки не должны выглядеть как ссылки.

## Text Rules

- Писать для человека без опыта программирования.
- H1 до 60 символов.
- Lead — до двух коротких предложений.
- Заголовок секции — до 45 символов.
- Описание карточки — одно короткое предложение.
- Если используется `ООП`, сразу пояснять: темы про классы и объекты.
- Не использовать рекламную воду, жаргон и обещания без функции.

## Visual And Motion Rules

- Использовать HTML/CSS/SVG без внешних изображений.
- Делать лёгкий маршрут, а не сетку одинаковых карточек.
- Hover/focus не меняют layout.
- `prefers-reduced-motion` отключает декоративные переходы.
- `HomePage.module.scss` не должен содержать `font-weight: 800`, `font-weight: 900`, `font-size: 64`, `font-size: 72`.

## Readiness Checklist

- `/` не внутри `AppLayout`.
- На `/` нет sidebar, search и AI assistant.
- Hero понятен новичку за 5 секунд.
- Карта маршрута объясняет первый путь до открытия кода.
- Нет повторов CTA и повторов одинакового смысла.
- Mobile без horizontal overflow.
- Light/dark/deep-dark читаемы.
