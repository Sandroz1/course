# План отдельного курса "База C++"

Документ описывает программу будущего курса "База C++". Это план для последующего наполнения разделов отдельными задачами и промптами.

## Что уже есть в проекте

- Тип `CourseId` поддерживает два курса: `base-cpp` и `oop-cpp`.
- В `site/src/data/courses.ts` курс "База C++" добавлен как отдельный курс и ведёт на `/courses/base-cpp`.
- Страница `/courses/base-cpp` показывает отдельный список разделов курса.
- `site/src/data/courseSections.ts` содержит отдельные секции для `base-cpp` и `oop-cpp`; ООП-разделы не смешиваются с базовым курсом.
- Ранние разделы `base-cpp` до условий помечены как `needs-theory`.
- Сейчас доступны разделы: `conditions`, `ternary-operator`, `switch-case`, `for-loop`, `do-while-loop`. Раздел `while-loop` пока добавлен как `needs-theory`.
- `site/src/data/tasks.ts` сейчас содержит задачи ООП-курса; практические задания `base-cpp` находятся внутри учебных разделов.
- `site/src/content/baseCpp` используется для контента "Базы C++", а `site/src/content/course` - для ООП-курса.

Вывод: `base-cpp` уже подключён как отдельный курс с частично готовыми разделами. Дальше нужно наполнять недостающие темы постепенно, не смешивая их с текущим ООП-контентом.

## Программа курса

### 0. Введение и инструменты

Цель: подготовить окружение и объяснить, как студент будет писать, запускать и проверять маленькие C++ программы.

Темы:
- что такое программа, исходный код и файл `.cpp`;
- компилятор, IDE, запуск консольной программы;
- базовый цикл работы: прочитать условие, написать код, скомпилировать, проверить;
- простые ошибки запуска: не тот файл, не тот проект, забытый `main`;
- вводная самопроверка базовых знаний.

Практика:
- открыть готовый `.cpp` файл;
- скомпилировать минимальную программу;
- исправить простую синтаксическую ошибку;
- вывести одну строку в консоль.

Основа из презентации: вводные слайды 1-8 и подготовительные слайды главы 2: 26-31. Задания презентации здесь не переносить целиком, чтобы раздел 0 не дублировал главу 2. Как лёгкую проверку можно использовать идею задания 2.1, но без разворачивания темы первой программы.

Предполагаемый файл контента: `site/src/content/baseCpp/introTools.ts`.

Сложность: самый простой раздел. Детали настройки IDE и флаги компилятора оставить как дополнение проекта, только если они нужны для запуска в текущей среде.

Переход к следующему разделу: после запуска простой программы можно обсуждать алгоритм как порядок действий до написания кода.

### 1. Алгоритмы

Цель: научить описывать решение задачи до кода.

Темы:
- алгоритм как последовательность действий;
- свойства алгоритмов: результативность, корректность, точность, понятность, дискретность, массовость;
- способы записи: текст, псевдокод, блок-схема;
- линейные, разветвлённые и циклические алгоритмы;
- входные данные, обработка, результат.

Практика:
- записать бытовой алгоритм по шагам;
- преобразовать текстовый алгоритм в псевдокод;
- нарисовать простую блок-схему;
- описать алгоритм с выбором и повторением;
- подготовить алгоритм для будущей программы.

Основа из презентации: глава 1, слайды 9-24. Задания 1.1-1.6 подходят для словесных алгоритмов и блок-схем, задания 1.7-1.8 про "Подброс монетки" подходят как переход от человеческого исполнителя к ЭВМ. При переносе убрать бытовые шутки и оставить учебную суть: последовательность действий, выбор, повторение, ввод и вывод.

Предполагаемый файл контента: `site/src/content/baseCpp/algorithms.ts`.

Сложность: базовая. Не уходить в сложность алгоритмов и оценку `O(...)`.

Переход к следующему разделу: первая программа C++ станет способом записать простой линейный алгоритм в коде.

### 2. Первая программа C++

Цель: разобрать минимальный каркас C++ программы и первый вывод на экран.

Темы:
- `#include <iostream>`;
- `using namespace std` на учебном уровне;
- `int main()`;
- фигурные скобки и точка с запятой;
- `return 0`;
- комментарии;
- `cout`;
- обычные строковые литералы и raw-строка как обзорная тема.

Практика:
- написать минимальную программу;
- вывести `Hello, World!`;
- вывести несколько строк информации о себе;
- исправить пропущенную точку с запятой или скобку;
- сравнить обычную строку и raw-строку на простом примере.

Основа из презентации: глава 2, слайды 25-35. Основные задания: 2.1 "Классическое" и 2.2 "Немного о себе". Слайды 26-31 использовать только как практический алгоритм создания и запуска программы, без повторного длинного блока про инструменты.

Предполагаемый файл контента: `site/src/content/baseCpp/firstProgram.ts`.

Сложность: базовая. Не углубляться в препроцессор и пространства имён.

Переход к следующему разделу: после вывода фиксированного текста нужны данные, переменные и операции.

### 3. Переменные, типы и операции

Цель: научить хранить значения и выполнять базовые вычисления.

Темы:
- переменная, имя, значение;
- объявление и инициализация;
- основные типы: `int`, `float`, `double`, `char`, `bool`;
- арифметические операции;
- присваивание;
- инкремент и декремент как краткая форма изменения значения;
- порядок вычислений на базовом уровне;
- область видимости как первое ограничение доступа к переменным.

Практика:
- объявить несколько переменных;
- посчитать сумму, разность и произведение;
- вычислить выражение по формуле;
- переставить порядок вывода значений;
- найти типичную ошибку в имени или области видимости переменной.

Основа из презентации: глава 2, слайды 36-40 и 42-46. Основные задания: 2.3 для вычисления формулы, 2.4 для суммы/разницы/произведения, 2.5 для работы с несколькими вещественными числами. Ввод из этих заданий можно использовать частично, основная цель раздела - переменные и операции.

Предполагаемый файл контента: `site/src/content/baseCpp/variables.ts`.

Сложность: базовая. Переполнение, сложные преобразования типов и `auto` не включать.

Переход к следующему разделу: чтобы значения не были зашиты в коде, нужен ввод и аккуратный вывод.

### 4. Ввод и вывод

Цель: научить получать данные от пользователя и показывать результат.

Темы:
- `cin` и `cout`;
- операторы `>>` и `<<`;
- приглашение ко вводу;
- ввод нескольких значений;
- вывод результата с понятной подписью;
- базовая ошибка ввода значения не того типа;
- простое форматирование текста без перегруза.

Практика:
- считать два числа и вывести сумму;
- считать значения для формулы и вывести результат;
- считать стороны прямоугольника и вывести площадь;
- вывести несколько введённых чисел в заданном порядке;
- оформить вывод так, чтобы ответ было легко проверить.

Основа из презентации: глава 2, слайд 41, задания 2.3-2.5. Эти задания здесь использовать в версии с обязательным вводом с клавиатуры и понятным выводом результата.

Предполагаемый файл контента: `site/src/content/baseCpp/inputOutput.ts`.

Сложность: базовая. Не добавлять сложную валидацию потоков, достаточно показать типичную ошибку ввода.

Переход к следующему разделу: после ввода данных программа должна выбирать разные действия по условиям.

### 5-7. Условия

Цель: научить выбирать ветку выполнения по условию.

В текущей реализации этот блок разбит на три раздела: `conditions`, `ternary-operator`, `switch-case`.

Темы:
- `if`, `else if`, `else`;
- операторы сравнения;
- логические выражения;
- вложенные условия на простом уровне;
- тернарный оператор как краткая форма выбора;
- `switch case` для меню;
- связь условия с пользовательским выбором.

Практика:
- вывести большее из двух чисел;
- расширить сравнение до трёх чисел;
- посчитать модули без библиотек;
- сделать простое меню через `if`;
- переписать меню через `switch case`.

Основа из презентации: глава 3, слайды 47-55. Основные задания: 3.1 для сравнения, 3.2 для модулей без функций и библиотек, 3.3 для меню через условия, 3.4 для меню через `switch case`.

Файлы контента: `site/src/content/baseCpp/conditions.ts`, `site/src/content/baseCpp/ternaryOperator.ts`, `site/src/content/baseCpp/switchCase.ts`.

Сложность: базовая с умеренным усложнением на меню. Не добавлять исключения и сложную обработку некорректного ввода.

Переход к следующему разделу: условия позволяют выбрать действие один раз, а циклы позволят повторять действия.

### 8-10. Циклы

Цель: научить повторять действия и накапливать результат.

В текущей реализации этот блок разбит на разделы `for-loop`, `while-loop`, `do-while-loop`. Раздел `while-loop` пока помечен как `needs-theory`.

Темы:
- общий смысл цикла;
- `for`;
- `while`;
- `do while`;
- счётчик и шаг цикла;
- сумма через переменную-контейнер;
- количество введённых значений;
- поиск минимума и максимума;
- корректная инициализация `min` и `max`.

Практика:
- вывести диапазоны чисел с разным шагом;
- вывести сообщение заданное количество раз;
- посчитать сумму фиксированного набора без цикла и затем с циклом;
- запрашивать числа до отрицательного значения;
- вывести диапазон и сумму чисел в нём;
- найти минимум, максимум и разницу.

Основа из презентации: глава 4, слайды 56-73. Основные задания: 4.1-4.2 для `for`, 4.3-4.5 для суммы и `while`, 4.6 для `min/max`. Слайд 62 обязательно использовать для `do while`, даже если отдельного задания на него нет.

Файлы контента: `site/src/content/baseCpp/forLoop.ts`, `site/src/content/baseCpp/whileLoop.ts`, `site/src/content/baseCpp/doWhileLoop.ts`.

Сложность: средняя для начинающего. Особенно внимательно объяснить сумму, счётчик и начальные значения минимума/максимума.

Переход к следующему разделу: повторяющийся код нужно выносить в функции.

### 7. Функции

Цель: научить выделять повторяемую логику в именованные блоки.

Темы:
- что такое функция;
- объявление и вызов функции;
- параметры;
- возвращаемое значение;
- `void`;
- область видимости между `main` и функцией;
- передача данных в функцию;
- простое разбиение программы на функции.

Практика:
- функция умножения двух чисел;
- функция вывода сообщения;
- функция вывода диапазона;
- функция модуля числа;
- функция суммы от 0 до введённого числа;
- функция степени через цикл.

Основа из презентации: глава 5, слайды 74-83. Основные задания: 5.1 для первых функций, 5.2 для функции модуля с возвращаемым значением, 5.3 для суммы, 5.4 для степени. Задание 5.5 использовать как дополнительную проверку условий и функций, хотя в презентации оно отмечено как не связанное напрямую с функциями.

Предполагаемый файл контента: `site/src/content/baseCpp/functions.ts`.

Сложность: средняя. Не добавлять перегрузку и шаблоны, они вынесены в раздел 12.

Переход к следующему разделу: функции помогут обрабатывать наборы данных, а для наборов нужны массивы.

### 8. Статические одномерные массивы

Цель: научить хранить и обрабатывать набор значений фиксированного размера.

Темы:
- статический массив и стек;
- индекс и размер массива;
- выход за границы массива как ошибка;
- обход массива циклом;
- заполнение массива вручную и программно;
- генерация случайных чисел;
- сумма, среднее, минимум, максимум;
- фильтрация элементов по условию.

Практика:
- создать массив на 3 элемента и вывести его;
- вывести массив в обратном порядке;
- показать, почему нельзя обращаться к несуществующим индексам;
- вывести чётные или нечётные элементы;
- заполнить массив с клавиатуры;
- посчитать сумму, среднее, минимум и максимум;
- выбрать диапазон случайного заполнения.

Основа из презентации: глава 6, слайды 84-94. Основные задания: 6.1 для базового массива и границ, 6.2 для фильтрации, 6.3 для суммы/среднего/min/max, 6.4 для случайного заполнения, 6.5 для фильтрации `float` значений.

Предполагаемый файл контента: `site/src/content/baseCpp/staticArrays.ts`.

Сложность: средняя. Ошибки выхода за границы показывать как предупреждение и разбор, а не как нормальную практику.

Переход к следующему разделу: двумерные массивы расширяют идею одномерного массива до таблиц и матриц.

### 9. Статические многомерные массивы

Цель: научить работать с таблицами фиксированного размера.

Темы:
- многомерный статический массив как массив массивов;
- строка, столбец, координаты элемента;
- представление двумерного массива как матрицы;
- вложенные циклы;
- вывод таблицы;
- сумма, минимум и максимум по всем элементам;
- поиск элемента и вывод координат.

Практика:
- создать массив `2 x 2` и вывести как матрицу;
- заполнить массив `5 x 8` случайными числами;
- посчитать сумму, минимум и максимум;
- вывести выбранную пользователем строку;
- найти все элементы с заданным значением;
- изменить элемент по координатам.

Основа из презентации: глава 7, слайды 95-105. Основные задания: 7.1 для ручной работы без циклов, 7.2 для вложенных циклов и агрегатов, 7.3 для строки и поиска по значению, 7.4 для изменения элемента по координатам.

Предполагаемый файл контента: `site/src/content/baseCpp/multidimensionalArrays.ts`.

Сложность: средняя. Трёхмерные массивы здесь не разворачивать, достаточно упомянуть как обобщение.

Переход к следующему разделу: для динамической памяти и передачи массивов нужно понимать адреса, указатели и ссылки.

### 10. Указатели и ссылки

Цель: объяснить адреса, динамические переменные, ссылки и передачу данных в функции.

Темы:
- адрес памяти;
- указатель как переменная с адресом;
- `nullptr`;
- разыменование;
- `new` и `delete`;
- утечка памяти;
- арифметика указателей;
- ссылка как псевдоним объекта;
- передача по ссылке в функцию;
- передача массивов в функции.

Практика:
- создать указатель и вывести его значение;
- разобрать ошибку разыменования `nullptr`;
- выделить и освободить динамическую переменную;
- изменить значение по адресу;
- пройти по массиву через арифметику указателей;
- изменить значение через функцию с ссылкой;
- передать одномерный и двумерный массив в функцию.

Основа из презентации: глава 8, слайды 106-127. Основные задания: 8.1-8.2 для указателей, `new` и `delete`, 8.3 для арифметики указателей, 8.4 для сравнения переменной, указателя и ссылки, 8.5 для передачи по ссылке, 8.6 для функций с массивами.

## Как курс "База C++" связан с текущим курсом "ООП C++"

- "База C++" должна идти перед "ООП C++".
- "ООП C++" не должен дублировать весь базовый курс.
- Раздел 0 в ООП можно оставить как краткое повторение и подготовку перед объектным подходом.
- Темы `this`, классы, конструкторы, инкапсуляция, наследование и полиморфизм остаются в ООП-курсе.
- Указатели, ссылки, функции, массивы, строки и файлы относятся к базе C++.
- В ООП-курсе можно ссылаться на базовый курс: если забыты массивы, функции или ссылки, повторить соответствующий раздел базы.
- Раздел 0 ООП не должен расширяться до полного курса основ, иначе он будет мешать основному фокусу на ООП.
