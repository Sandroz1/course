# План отдельного курса "База C++"

Документ описывает программу будущего курса "База C++". Это план для последующего наполнения разделов отдельными задачами и промптами.

## Что уже есть в проекте

- Тип `CourseId` уже поддерживает два курса: `base-cpp` и `oop-cpp`.
- В `site/src/data/courses.ts` курс "База C++" уже есть как `status: "soon"` и ведёт на `/courses/base-cpp`.
- `BaseCppCoursePage` сейчас является заглушкой с коротким списком планируемых тем.
- `site/src/data/courseSections.ts` фактически содержит только разделы курса "ООП C++": все секции мапятся в `courseId: "oop-cpp"`.
- `site/src/data/tasks.ts` сейчас создаёт задачи с `courseId: "oop-cpp"`.
- `site/src/content/course` сейчас используется для контента ООП-курса, включая раздел 0 "Подготовка".

Вывод: проект частично готов к нескольким курсам на уровне типов и карточек курсов, но полноценная маршрутизация уроков и задач сейчас фактически рассчитана на один основной список `courseSections` для "ООП C++". Для "База C++" лучше планировать отдельный набор секций и отдельный каталог контента, например `site/src/content/baseCpp`.

## Источник и границы

Источник программы: презентация "Основы программирования на языке C++" и текущая структура проекта.

Файл презентации в репозитории не найден, поэтому в плане не указаны точные номера слайдов или заданий. В колонке "задания из презентации" перечислены типы заданий, которые нужно сверить с исходной презентацией при написании конкретного раздела.

Курс "База C++" не должен заменять раздел 0 текущего курса "ООП C++". Раздел 0 в ООП остаётся краткой подготовкой перед ООП, а полноценная база выносится в отдельный курс.

## Программа курса

### 0. Введение и инструменты

Предполагаемый файл: `site/src/content/baseCpp/introTools.ts`

Цель раздела: подготовить окружение и объяснить, как студент будет писать, запускать и проверять маленькие C++ программы.

Основные темы:
- что такое программа и исходный код;
- компилятор, IDE, файл `.cpp`;
- как запустить одну консольную программу;
- базовая структура работы: прочитать условие, написать код, скомпилировать, проверить;
- типичные ошибки запуска: не тот файл, не тот проект, забытый `main`.

Практические задачи:
- открыть готовый `.cpp` файл;
- скомпилировать минимальную программу;
- намеренно исправить простую синтаксическую ошибку;
- вывести одну строку в консоль.

Задания из презентации как основа: вводные задания на запуск первой программы и знакомство с инструментами.

Комментарий по сложности: самый простой раздел; не нужно перегружать настройками IDE и флагами компилятора.

Связь со следующим разделом: после запуска простых программ можно обсуждать алгоритм как порядок действий до написания кода.

### 1. Алгоритмы

Предполагаемый файл: `site/src/content/baseCpp/algorithms.ts`

Цель раздела: научить описывать решение задачи до кода.

Основные темы:
- алгоритм как последовательность шагов;
- входные данные, обработка, результат;
- линейный алгоритм;
- ветвление на уровне идеи;
- повторение действий на уровне идеи;
- простая блок-схема или псевдокод без формального перегруза.

Практические задачи:
- записать алгоритм приготовления простого действия в шагах;
- описать алгоритм вычисления суммы двух чисел;
- описать алгоритм проверки чётности;
- перевести короткий псевдокод в будущую структуру программы.

Задания из презентации как основа: задания на составление алгоритмов, блок-схемы, поиск входных и выходных данных.

Комментарий по сложности: важно не уходить в теорию алгоритмической сложности; раздел должен быть прикладным.

Связь со следующим разделом: первая программа C++ станет способом записать простой линейный алгоритм в коде.

Практические задачи:

Задания из презентации как основа: первая программа, вывод текста, разбор структуры `main`.

Комментарий по сложности: не объяснять глубоко препроцессор и namespaces; эти детали достаточно назвать и использовать.

Связь со следующим разделом: после каркаса нужна работа с данными: переменные, типы и операции.

### 3. Переменные, типы и операции

Задания из презентации как основа: вычислительные задачи с формулами, работа с целыми и вещественными числами.

Комментарий по сложности: не вводить сложные преобразования типов и переполнение глубоко; достаточно показать типичные ошибки.

Связь со следующим разделом: чтобы переменные были не только заданными в коде, студенту нужен ввод и вывод.

- проверить простое условие по строке;
- заменить или найти символ на базовом уровне.

Задания из презентации как основа: задания на ввод строк, обработку символов, подсчёт, поиск.

Комментарий по сложности: аккуратно объяснить `getline` после `cin`, но не углубляться в кодировки и сложную обработку текста.

Связь со следующим разделом: строки и массивы позволяют читать и записывать данные в файлы.

### 15. Файлы

Предполагаемый файл: `site/src/content/baseCpp/files.ts`

Практические задачи:

Связь со следующим разделом: после файлов студент готов переходить к ООП C++, где данные и действия будут объединяться в структуры и классы.

## Как курс "База C++" связан с текущим курсом "ООП C++"

- "База C++" должна идти перед "ООП C++".
- "ООП C++" не должен дублировать весь базовый курс.
- Раздел 0 в ООП можно оставить как краткое повторение и подготовку перед объектным подходом.
- Темы `this`, классы, конструкторы, инкапсуляция, наследование и полиморфизм остаются в ООП-курсе.
- Указатели, ссылки, функции, массивы, строки и файлы относятся к базе C++.
- В ООП-курсе можно ссылаться на базовый курс: "если забыли массивы/функции/ссылки, повторите разделы базы".
- Раздел 0 ООП не должен расширяться до полного курса основ; иначе он станет слишком большим и будет мешать основному фокусу на ООП.
