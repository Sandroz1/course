# План тем курса ООП C++

Документ фиксирует рабочую карту курса: что уже есть в проекте, что нужно расширить, что нужно создать позже.

## Источники

- Презентация: `ООП C++ (главы 1-9).pdf`.
- Текущие темы: `site/src/content/course`.
- Текущий порядок: `site/src/data/courseSections.ts`.
- Эталон готовности: `basics.ts`, `oop.ts`, `structs.ts`, `classes.ts`, `constructorsDestructors.ts`, `thisKeyword.ts`.

## Принцип порядка

Текущий порядок разделов 0-12 в проекте совпадает с предлагаемым порядком и его стоит сохранить. Он лучше механического порядка презентации для новичка: сначала вводится базовый C++, затем идея ООП, структуры, классы, жизненный цикл объекта и `this`; только после этого студент переходит к многофайловости, спискам инициализации, `vector`, инкапсуляции и исключениям.

Раздел `this` остаётся разделом 5, хотя в презентации он находится внутри главы 3. Это правильно: студенту сначала нужны классы, конструкторы и понимание текущего объекта.

Главы 10-12 есть в оглавлении презентации, но подробные слайды в найденном PDF не представлены. Поэтому разделы 21-24 лучше планировать позже, после завершения тем 13-20.

## Критерии готовности темы

- Тема объясняет одну основную идею, а не несколько несвязанных возможностей C++.
- В начале есть понятная проблема, которую тема решает.
- Примеры короткие и связаны с текущими задачами курса.
- Есть плохой пример и исправленный вариант.
- Есть блок "когда не использовать", чтобы студент не применял приём везде.
- Итоговая практика связана с уже существующими задачами или будущей большой задачей.
- Тема не меняет API frontend/backend и не требует новой зависимости.
