## Большой синтаксический тренажёр перед курсом

Этот раздел можно читать как справочник. Он объясняет не только «что такое структура или класс», а именно «как это написать руками»: где ставится `;`, зачем нужны `{}`, `()`, `<>`, `::`, `.`, `->`, `&`, `:` и почему компилятор ругается, если один символ пропущен.

### 1. Базовая программа C++

**Шаблон:**

```cpp
#include <iostream>

using namespace std;

int main() {
    return 0;
}
```

**Расшифровка:**

- `#include` — команда препроцессора. Она говорит: перед компиляцией подключи другой файл.
- `<iostream>` — стандартная библиотека для `cin` и `cout`. Угловые скобки `< >` используют для стандартных библиотек C++.
- `using namespace std;` — разрешает писать `cout`, `cin`, `string`, `vector` без приставки `std::`.
- После `using namespace std` ставится `;`, потому что это инструкция C++.
- `int main()` — главная функция программы. С неё начинается запуск.
- `int` перед `main` означает, что функция возвращает целое число.
- `()` после `main` нужны всегда: функция вызывается через круглые скобки.
- `{}` показывают тело функции: код, который относится к `main`.
- `return 0;` возвращает число `0`, то есть программа завершилась успешно.

**Плохой пример:**

```cpp
#include iostream

int main {
    return 0
}
```

Ошибки: нет `< >` вокруг `iostream`, нет `()` после `main`, нет `;` после `return 0`.

### 2. Переменные

**Шаблон:**

```cpp
тип имяПеременной;
тип имяПеременной = значение;
```

Сначала пишется тип, потом имя, потом при необходимости `= значение`, потом обязательно `;`.

```cpp
int age;
double price;
string name;
bool isStudent;

int count = 5;
double cost = 199.99;
string city = "Moscow";
bool isOpen = true;
```

Частая ошибка:

```cpp
age int; // неверный порядок
int age  // нет ;
```

### 3. Ввод и вывод

```cpp
cout << "Текст";
cin >> variable;
```

- `cout` выводит на экран.
- `<<` можно читать как «отправить в вывод».
- `cin` читает с клавиатуры.
- `>>` можно читать как «забрать из ввода в переменную».
- Если перепутать стрелки, код либо не скомпилируется, либо будет означать другое действие.

Вывод нескольких значений:

```cpp
cout << "Возраст: " << age << '\n';
```

`endl` и `'\n'` переводят строку:

```cpp
cout << "Привет" << endl;
cout << "Привет\n";
cout << "Привет" << '\n';
```

Для учебных задач `'\n'` обычно проще и быстрее.

### 4. Условия

**Шаблон:**

```cpp
if (условие) {
    код;
} else {
    код;
}
```

- После `if` всегда идут круглые скобки `()`.
- В круглых скобках пишется условие: `age >= 18`.
- В фигурных скобках `{}` пишется код, который выполнится при истинном условии.
- `else` ставится после закрывающей `}` блока `if`.
- После `if (...)` точка с запятой не нужна.

Плохой пример:

```cpp
if (age >= 18);
{
    cout << "Можно";
}
```

Ошибка в `;` после `if`. Эта точка с запятой делает условие пустым, а блок `{ cout << "Можно"; }` выполнится отдельно почти всегда.

### 5. Циклы

**`for`:**

```cpp
for (int i = 0; i < count; i++) {
    код;
}
```

- `int i = 0` — старт счётчика.
- `i < count` — условие продолжения.
- `i++` — изменение счётчика после каждого прохода.
- Части внутри `for (...)` разделяются `;`, потому что это три разные части настройки цикла.

**`while`:**

```cpp
while (условие) {
    код;
}
```

`while` читается как «пока условие истинно, выполняй код».

### 6. Функции

**Шаблон:**

```cpp
типВозврата имяФункции(тип параметр) {
    return значение;
}
```

- Тип возврата говорит, что функция отдаёт наружу: `int`, `double`, `string`.
- Имя функции пишется в camelCase: `getSquare`.
- Параметры пишутся в круглых скобках.
- Тело функции пишется в `{}`.
- `return` возвращает результат.
- Если функция ничего не возвращает, пишут `void`.

```cpp
int getSquare(int number) {
    return number * number;
}

void printHello() {
    cout << "Hello";
}
```

### 7. Структуры

```cpp
struct Person {
    string name;
    int age;
};
```

- `struct` создаёт новый тип.
- `Person` — имя типа.
- В `{}` пишутся поля.
- Каждое поле: сначала тип, потом имя, потом `;`.
- После закрывающей `}` структуры обязательно ставится `;`.

Создание объекта и обращение к полю:

```cpp
Person person;
person.name = "Anna";
```

- `Person` — тип.
- `person` — имя объекта.
- `.` означает «взять поле внутри объекта».

Передача в функцию:

```cpp
void printPerson(const Person& person) {
    cout << person.name;
}
```

- `const` означает «функция не будет менять объект».
- `Person&` означает «передаём по ссылке, без копирования».
- `&` относится к типу параметра: это ссылка на существующий объект.

### 8. Классы

```cpp
class Car {
private:
    string brand;

public:
    void print() {
        cout << brand;
    }
};
```

- `class` создаёт новый тип.
- `Car` — имя класса.
- `private:` — всё ниже закрыто снаружи, пока не встретится другой модификатор доступа.
- `public:` — всё ниже доступно снаружи.
- После `private` и `public` ставится `:`, потому что это метки областей доступа.
- После класса обязательно `;`.

```cpp
Car car;
```

`Car` — тип, `car` — объект.

### 9. Поля и методы

Поле:

```cpp
тип имяПоля;
```

Метод:

```cpp
типВозврата имяМетода(параметры) {
    код;
}
```

Метод отличается от обычной функции тем, что написан внутри класса или реализован как метод класса через `ClassName::methodName`.

### 10. Конструкторы

```cpp
ClassName() {
    код;
}

ClassName(тип параметр) {
    поле = параметр;
}
```

- У конструктора нет `void`, `int` или другого типа возврата.
- Имя конструктора полностью совпадает с именем класса.
- Конструктор вызывается при создании объекта.
- Какой конструктор вызовется, зависит от аргументов.

Ошибка:

```cpp
void Student() {
}
```

Это не конструктор, а обычный метод с именем `Student`, потому что написан `void`.

### 11. Деструктор

```cpp
~ClassName() {
    код;
}
```

- `~` показывает, что это деструктор.
- У деструктора нет типа возврата.
- У деструктора нет параметров.
- Деструктор только один, потому что объект уничтожается одним способом.

### 12. Указатель `this`

```cpp
this->field = field;
```

- `this` — указатель на текущий объект.
- `->` используется для обращения к полю через указатель.
- `this->name` — поле объекта.
- `name` без `this->` может быть параметром функции.

Плохо:

```cpp
void setName(string name) {
    name = name;
}
```

Хорошо:

```cpp
void setName(string name) {
    this->name = name;
}
```

### 13. Список инициализации конструктора

```cpp
ClassName() : field1(value1), field2(value2) {
}
```

- `:` ставится после круглых скобок конструктора и до тела `{}`.
- Поля перечисляются через `,`.
- Значения пишутся в круглых скобках.
- Тело конструктора начинается после списка инициализации.

```cpp
Plate() : height(0), width(0), price(0) {}
```

Это означает: поля `height`, `width`, `price` создаются сразу со значением `0`.

### 14. Делегирование конструкторов

```cpp
ClassName(параметры) : ClassName(другиеАргументы) {
}
```

Один конструктор вызывает другой. Если конструктор делегирует, он не может в этом же списке инициализации перечислять поля.

Чтение слева направо:

```cpp
Worker(int id, string name) : Worker(id, name, "Не указан", 0) {}
```

«Конструктор с `id` и `name` вызывает другой конструктор `Worker` с четырьмя аргументами».

### 15. Геттеры и сеттеры

Геттер:

```cpp
тип getField() const {
    return field;
}
```

- Геттер возвращает значение поля.
- `const` после `()` обещает не менять объект.
- `return` отдаёт значение наружу.

Сеттер:

```cpp
void setField(тип field) {
    this->field = field;
}
```

- Сеттер часто `void`, потому что он меняет объект, а не возвращает результат.
- Параметр нужен, чтобы передать новое значение.
- `this` нужен, если имя параметра совпадает с именем поля.

### 16. Модификаторы доступа

```cpp
private:
public:
protected:
```

После модификаторов доступа ставится `:`, потому что это не инструкция, а метка раздела класса.

Область действия продолжается до следующего модификатора доступа или до конца класса.

Обычно поля пишут в `private`, а методы для работы с ними — в `public`.

### 17. `explicit`

```cpp
explicit ClassName(тип параметр);
```

`explicit` ставится перед конструктором с одним параметром, чтобы запретить неявное превращение значения в объект.

Без `explicit`:

```cpp
class Box {
public:
    Box(int size) {}
};
```

С `explicit`:

```cpp
class Box {
public:
    explicit Box(int size) {}
};
```

### 18. `std::vector`

```cpp
vector<тип> имя;
```

- `vector` — динамический массив.
- `<тип>` говорит, какие элементы хранятся внутри.
- Угловые скобки `< >` используются для типа элемента.

```cpp
vector<int> numbers;
vector<string> names;
vector<Car> cars;
```

Добавление:

```cpp
items.push_back(value);
```

Удаление:

```cpp
items.erase(items.begin() + index);
```

Размер:

```cpp
items.size()
```

Цикл:

```cpp
for (int i = 0; i < static_cast<int>(items.size()); i++) {
    cout << items[i];
}
```

- `items[i]` — элемент по индексу.
- Индексы начинаются с `0`.
- Номер пользователя часто переводят в индекс через `index - 1`.
- `static_cast<int>` простыми словами: «считай размер как `int`, чтобы удобно сравнивать с переменной `i`».

### 19. Range-based for

```cpp
for (const Type& item : items) {
    item.print();
}
```

- `const` — не меняем элемент.
- `Type&` — ссылка на элемент без копирования.
- `item` — имя одного текущего элемента.
- `:` читается как «из».
- `items` — контейнер.

Вся строка читается: «для каждого `item` типа `Type` из `items` выполнить код».

### 20. `std::sort`

```cpp
sort(items.begin(), items.end(), compareFunction);
```

- `items.begin()` — начало контейнера.
- `items.end()` — позиция после последнего элемента.
- `compareFunction` — функция сравнения.
- Нужен `#include <algorithm>`.

Функция сравнения:

```cpp
bool compareCarsByYear(const Car& left, const Car& right) {
    return left.getYear() < right.getYear();
}
```

- `bool` нужен, потому что функция отвечает «да» или «нет».
- `left` и `right` — два объекта, которые сортировка сравнивает.
- `return left.getYear() < right.getYear();` означает: левый автомобиль должен стоять раньше, если его год меньше.

### 21. Многофайловый проект

`Bed.hpp`:

```cpp
#pragma once

#ifndef BED_HPP
#define BED_HPP

class Bed {
public:
    void print() const;
};

#endif
```

`Bed.cpp`:

```cpp
#include "Bed.hpp"

void Bed::print() const {
}
```

`main.cpp`:

```cpp
#include "Bed.hpp"

int main() {
    Bed bed;
    bed.print();
    return 0;
}
```

- В `.hpp` объявление: что существует.
- В `.cpp` реализация: как работает.
- `#include "Bed.hpp"` подключает свой файл.
- Для своих файлов используют `" "`, для стандартных библиотек — `< >`.
- `Bed::print` означает «метод `print` класса `Bed`».
- `::` — оператор области видимости.
- Если забыть `Bed::`, получится обычная функция `print`, а не метод класса.

### 22. Include guard

```cpp
#ifndef BED_HPP
#define BED_HPP

// код

#endif
```

- `#ifndef BED_HPP` — если имя `BED_HPP` ещё не определено.
- `#define BED_HPP` — определить имя `BED_HPP`.
- `#endif` — закончить условный блок.
- Имя пишут большими буквами по традиции, чтобы было видно, что это макрос.
- `#pragma once` делает похожую защиту одной строкой; в учебной задаче можно оставить и его, и guard.

### 23. Исключения

```cpp
throw runtime_error("Ошибка");
```

```cpp
try {
    код;
} catch (const exception& error) {
    cout << error.what();
}
```

- `throw` бросает исключение.
- `runtime_error` — стандартная ошибка выполнения.
- Строка `"Ошибка"` описывает проблему.
- `try` содержит код, где ошибка может возникнуть.
- `catch` ловит ошибку.
- `const exception& error` — принимаем ошибку по ссылке и не меняем её.
- `error.what()` возвращает текст ошибки.

### 24. Меню в консольной программе

```cpp
int choice = 0;

while (choice != 5) {
    cout << "1. Показать\n";
    cout << "2. Добавить\n";
    cout << "3. Удалить\n";
    cout << "4. Редактировать\n";
    cout << "5. Выйти\n";
    cin >> choice;

    if (choice == 1) {
        // действие
    } else if (choice == 2) {
        // действие
    } else if (choice == 3) {
        // действие
    } else if (choice == 4) {
        // действие
    } else if (choice == 5) {
        cout << "Выход\n";
    } else {
        cout << "Неверный выбор\n";
    }
}
```

- Меню делается через цикл, потому что пользователь может выбирать действия много раз.
- `choice` объявляют до цикла, потому что условие `while` должно видеть эту переменную.
- `if / else if / else` выбирает действие.
- Выход происходит, когда `choice` становится равным `5`, и условие `choice != 5` становится ложным.

**IDE** — среда разработки. Это программа, где удобно писать код, запускать его и видеть ошибки. Подойдут Visual Studio, CLion, Code::Blocks или VS Code с расширениями.

### Как устроена простая программа C++

```cpp
#include <iostream>

using namespace std;

int main() {
    cout << "Привет, C++!" << endl;
    return 0;
}
```

Разберём блоками.

`#include <iostream>` подключает библиотеку для ввода и вывода. Без неё `cout` и `cin` неизвестны компилятору.

`using namespace std;` позволяет писать `cout`, а не `std::cout`. Это упрощение для первых программ.

`int main()` — главная функция. Именно с неё начинается выполнение программы.

`cout << "Привет, C++!" << endl;` выводит текст на экран. `endl` переводит строку.

`return 0;` сообщает операционной системе, что программа завершилась успешно.

### Ввод и вывод

```cpp
#include <iostream>
#include <string>

    string name;
    int age;

    cout << "Введите имя: ";
    cin >> name;

    cout << "Введите возраст: ";

    return 0;
}
```

`cin >> name;` читает одно слово. Если нужно читать строку с пробелами, используют `getline`, но в задачах курса для простоты можно вводить значения без пробелов или заменять пробелы подчёркиваниями.

### Переменная

**Переменная** — именованная ячейка памяти, где хранится значение.

```cpp
int age = 18;
double price = 199.99;
string name = "Anna";
bool isStudent = true;
```

`int` хранит целые числа. `double` хранит дробные числа. `string` хранит текст. `bool` хранит `true` или `false`.

### Функция

**Функция** — кусок программы с именем. Она нужна, чтобы не писать один и тот же код много раз.

```cpp
int square(int number) {
    return number * number;
}
```

`int square(...)` означает: функция называется `square` и возвращает целое число.

`int number` — параметр функции. Это значение, которое функция получает снаружи.

`return` возвращает результат.

### Условие

**Условие** позволяет выполнить один код, если проверка истинна, и другой код, если проверка ложна.

```cpp
if (age >= 18) {

    cout << "Несовершеннолетний";
}
```

### Цикл

**Цикл** повторяет действия несколько раз.

```cpp
for (int i = 0; i < 5; i++) {
    cout << i << endl;
}
```

`int i = 0` — начальное значение. `i < 5` — пока это правда, цикл работает. `i++` увеличивает `i` на 1 после каждого прохода.

### Массив

**Массив** — несколько значений одного типа под одним именем.

```cpp
int numbers[3] = {10, 20, 30};
```

Индексы начинаются с нуля. У массива из 3 элементов индексы: `0`, `1`, `2`.

## Раздел 1. Что такое ООП

### Синтаксис темы

Этот блок нужен не для идеи, а для ручного письма кода: какие слова писать, какие скобки ставить и где заканчивается объявление.

**Общий шаблон:**

```cpp
class ИмяКласса {
public:
    тип поле;
    типВозврата метод() {
        код;
    }
};

ИмяКласса object;
```

**Расшифровка шаблона:**

- `class` создаёт новый тип.
- Имя класса пишут с большой буквы.
- В `{}` пишется содержимое класса.
- `public:` открывает доступ к полям и методам ниже.
- После класса обязательно `;`.
- Объект создаётся как `Тип имя;`.

**Мини-пример:**

```cpp
class Student {
public:
    string name;

    void print() {
        cout << name << '\n';
    }
```

**Разбор мини-примера по строкам:**

- `public:` делает `name` и `print` доступными снаружи;
- `string name;` объявляет поле;
- `void print()` объявляет метод без возвращаемого значения;
- `Student student;` создаёт объект.

**Частые ошибки в синтаксисе:**

- забыли `;` после класса;
- написали объект до объявления класса;
- забыли `()` при вызове метода: нужно `student.print();`;
- перепутали класс `Student` и объект `student`.

ООП — объектно-ориентированное программирование. Это не просто синтаксис C++. Это способ думать о программе.

В процедурном подходе программа выглядит как набор функций:

```cpp
string carBrand;
int carYear;

void printCar(string brand, int year) {
    cout << brand << " " << year << endl;
}
```

Данные отдельно, функции отдельно. Пока программа маленькая, это удобно. Когда данных становится много, легко перепутать, какая функция с какими данными работает.

В ООП мы объединяем данные и действия в объект.

```cpp
class Car {
public:
    string brand;
    int year;

    void print() {
        cout << brand << " " << year << endl;
    }
};
```

Теперь автомобиль хранит свои данные и умеет сам себя вывести.

### Основные понятия

**Класс** — описание будущих объектов. Можно представить как чертёж.

**Объект** — конкретная вещь, созданная по классу. Если `Car` — чертёж автомобиля, то `Car toyota` — конкретный автомобиль.

**Поле** — переменная внутри класса. Поля описывают состояние объекта.

**Метод** — функция внутри класса. Методы описывают поведение объекта.

**Состояние объекта** — текущие значения его полей.

**Поведение объекта** — то, что объект умеет делать через методы.

### Пример: студент

```cpp
class Student {
public:
    string name;
    int age;

    void print() {
        cout << "Имя: " << name << ", возраст: " << age << endl;
    }
};

int main() {
    Student student;
    student.name = "Анна";
    student.age = 18;
    student.print();

    return 0;
}
```

`Student student;` создаёт объект. `student.name` обращается к полю. `student.print()` вызывает метод.

### Примеры из реального мира

Автомобиль: поля — марка, год, пробег; методы — вывести информацию, изменить пробег.

Студент: поля — имя, возраст, id, группа; методы — вывести, отредактировать.

Планета: поля — название, масса, диаметр; методы — вывести информацию.

Компания: поля — доходы, расходы, тип; методы — изменить доходы, получить прибыль.

```cpp
struct Person {
```

### Синтаксис темы

Этот блок нужен не для идеи, а для ручного письма кода: какие слова писать, какие скобки ставить и где заканчивается объявление.

**Общий шаблон:**

```cpp
class ИмяКласса {
private:
    тип поле;

public:
    типВозврата метод();
};
```

**Расшифровка шаблона:**

- `private:` закрывает поля от прямого доступа.
- `public:` открывает методы для внешнего кода.
- После `private` и `public` ставится `:`.
- Метод внутри класса может обращаться к закрытым полям.
- После объявления класса ставится `;`.

**Мини-пример:**

```cpp
class Car {
private:
    string brand;

public:
    void print() {
        cout << brand;
    }
};
```

**Разбор мини-примера по строкам:**

- поле `brand` закрыто;
- метод `print` открыт;
- внутри `print` можно читать `brand`, потому что метод принадлежит классу;
- снаружи нужно вызывать `car.print()`.

**Частые ошибки в синтаксисе:**

- пытаются написать `car.brand`, когда `brand` закрыт;
- забывают `public:` и методы остаются закрытыми;
- забывают `;` после класса;
- пишут `private;` вместо `private:`.

### Синтаксис темы

Этот блок нужен не для идеи, а для ручного письма кода: какие слова писать, какие скобки ставить и где заканчивается объявление.

**Общий шаблон:**

```cpp
class ИмяКласса {
public:
    ИмяКласса();
    ИмяКласса(тип параметр);
    ~ИмяКласса();
};
```

**Расшифровка шаблона:**

- Конструктор называется как класс.
- У конструктора нет типа возврата, даже `void`.
- Деструктор начинается с `~`.
- У деструктора нет параметров.
- Объявления методов внутри класса заканчиваются `;`.

**Мини-пример:**

```cpp
class Student {
public:
    Student() {
        cout << "Создан";
    }

    ~Student() {
        cout << "Удалён";
    }
};
```

**Разбор мини-примера по строкам:**

- `Student()` вызывается при создании объекта;
- `~Student()` вызывается при уничтожении;
- оба метода не имеют типа возврата;
- после реализации внутри класса отдельная `;` после метода не нужна, но после класса нужна.

**Частые ошибки в синтаксисе:**

- написали `void Student()`;
- написали деструктор с параметром;
- забыли `~`;
- ожидают, что деструктор можно перегрузить.

**Конструктор** — специальный метод, который вызывается при создании объекта.

```cpp
class Student {
public:
    Student() {
        cout << "Студент создан" << endl;

```

Конструктор называется так же, как класс, и не имеет типа возвращаемого значения.

### Конструктор по умолчанию

Конструктор без параметров:

```cpp
Student() {
    name = "Без имени";
    age = 0;
}
```

### Конструктор с параметрами

```cpp
Student(string name, int age) {
    this->name = name;
    this->age = age;
}
```

Параметры позволяют сразу создать объект с нужными данными.

### Перегрузка конструкторов

Перегрузка означает: несколько функций с одним именем, но разными параметрами.

```cpp
Student() {}
Student(string name) {}
Student(string name, int age) {}
```

Компилятор сам выбирает нужный конструктор по переданным аргументам.

### Деструктор

**Деструктор** — метод, который вызывается при уничтожении объекта.

```cpp
~Student() {
    cout << "Студент удалён" << endl;
}
```

Деструктор называется как класс, но перед именем стоит `~`. У деструктора нет параметров и он всегда один.

Почему один? Потому что уничтожение объекта происходит одним способом: объект исчезает, а не создаётся в разных вариантах.

Когда деструктор реально нужен? Когда объект владеет ресурсом: файлом, памятью, подключением. В учебной задаче он нужен, чтобы увидеть момент удаления объекта.

### Как работает `vector<Student>`

`vector` хранит много объектов подряд. `students.push_back(readStudent())` добавляет нового студента в конец. Когда программа завершается, `vector` уничтожается, а вместе с ним уничтожаются студенты. Поэтому деструктор печатает сообщения при выходе.

### Синтаксис темы

Этот блок нужен не для идеи, а для ручного письма кода: какие слова писать, какие скобки ставить и где заканчивается объявление.

**Общий шаблон:**

```cpp
#pragma once

#ifndef ИМЯ_HPP
#define ИМЯ_HPP

// код

#endif
```

**Расшифровка шаблона:**

- `#pragma once` просит подключить файл один раз.
- `#ifndef` проверяет, не определён ли макрос.
- `#define` определяет макрос.
- `#endif` закрывает блок.
- Имя guard обычно пишут большими буквами.

**Мини-пример:**

```cpp
#pragma once

#ifndef BED_HPP
#define BED_HPP

class Bed {
};

#endif
```

**Разбор мини-примера по строкам:**

- строка 1 — короткая защита;
- строки 3–4 — классическая защита;
- `class Bed` находится внутри защиты;
- `#endif` закрывает `#ifndef`.

**Частые ошибки в синтаксисе:**

- забыли `#endif`;
- разные имена в `#ifndef` и `#define`;
- закрыли guard до объявления класса;
- удалили `#pragma once`, хотя его можно оставить.

**Препроцессор** — этап обработки кода до основной компиляции. Он выполняет команды, начинающиеся с `#`.

`#include` подключает другой файл.

`#define` создаёт имя-макрос.

`#ifndef` читается как «если не определено».

`#endif` завершает блок препроцессора.

### Зачем нужна защита

Если один и тот же `.hpp` случайно подключится несколько раз, компилятор может увидеть класс дважды и выдать ошибку повторного определения.

Классическая защита:

```cpp
#ifndef BED_HPP
#define BED_HPP

class Bed {
    // ...
};

#endif
```

Современная короткая защита:

```cpp
#pragma once
```

В задаче 4.3 просят использовать `#ifndef` и `#define`, но не удалять `#pragma once`, если он уже есть.

`std::vector` — динамический массив. Простыми словами: это список элементов одного типа, который умеет сам увеличиваться.

### Синтаксис темы

Этот блок нужен не для идеи, а для ручного письма кода: какие слова писать, какие скобки ставить и где заканчивается объявление.

**Общий шаблон:**

```cpp
ClassName(параметры) : ClassName(другиеАргументы) {
}
```

**Расшифровка шаблона:**

- После `:` вызывается другой конструктор того же класса.
- Внутри скобок передаются аргументы.
- Тело текущего конструктора выполняется после вызова делегируемого.
- Нельзя в том же списке одновременно писать поля `field(value)`.

**Мини-пример:**

```cpp
Worker(int id, string name) : Worker(id, name, "Не указан", 0) {}
```

**Разбор мини-примера по строкам:**

- объявлен конструктор с двумя параметрами;
- после `:` вызывается полный конструктор `Worker`;
- недостающие значения заменены стандартными;
- тело пустое, потому что работа уже сделана полным конструктором.

**Частые ошибки в синтаксисе:**

- пытаются вызвать конструктор в теле: `Worker(id, name);`;
- создают бесконечную цепочку;
- одновременно пишут `: Worker(...), id(id)`;
- забывают полный конструктор.

**Делегирование конструкторов** — ситуация, когда один конструктор вызывает другой конструктор того же класса.

```cpp
class Worker {
public:
    Worker(string name) : Worker(name, "Не указан") {}

    Worker(string name, string department) {
        // полный код инициализации
    }
};
```

Зачем это нужно? Чтобы не повторять одинаковую инициализацию в каждом конструкторе.

Важно: список инициализации и делегирование используют место после двоеточия. Поэтому конструктор либо делегирует другому, либо сам инициализирует поля.

```cpp
Worker(string name) : Worker(name, "Не указан") {}
```

Здесь конструктор с одним параметром вызывает конструктор с двумя параметрами.

### Синтаксис темы

Этот блок нужен не для идеи, а для ручного письма кода: какие слова писать, какие скобки ставить и где заканчивается объявление.

**Общий шаблон:**

```cpp
тип getField() const {
    return field;
}

void setField(тип field) {
    this->field = field;
}

explicit ClassName(тип value);
```

**Расшифровка шаблона:**

- Геттер возвращает значение и часто имеет `const` после `()`.
- Сеттер принимает новое значение и часто ничего не возвращает.
- `this->field` нужен при совпадении имён.
- `explicit` ставят перед конструктором с одним параметром.

**Мини-пример:**

```cpp
string getName() const {
    return name;
}

void setName(string name) {
    this->name = name;
}
```

**Разбор мини-примера по строкам:**

- `string` перед `getName` — тип возвращаемого значения;
- `const` после скобок запрещает менять объект;
- `return name;` отдаёт поле;
- `void setName` меняет поле и ничего не возвращает.

**Частые ошибки в синтаксисе:**

- забыли `return` в геттере;
- забыли `const` у геттера;
- сеттер возвращает значение без необходимости;
- пишут `name = name;`.

**Инкапсуляция** — идея, что объект сам хранит свои данные и сам контролирует доступ к ним.

Поля обычно делают `private`, чтобы внешний код не мог записать в объект неправильные данные.

```cpp
class Planet {
private:
    double diameter;

public:
    void setDiameter(double newDiameter) {
        if (newDiameter >= 0) {
            diameter = newDiameter;
        }
    }

    double getDiameter() const {
        return diameter;
    }
};
```

**Геттер** — метод для получения значения. Обычно начинается с `get`.

**Сеттер** — метод для изменения значения. Обычно начинается с `set`.

Сеттер может проверять данные. Например, диаметр планеты не должен быть отрицательным.

### `public`, `private`, `protected`

`public` — доступно всем.

`private` — доступно только внутри класса.

`protected` — доступно внутри класса и в наследниках. Наследование в этом курсе не изучаем глубоко, поэтому пока достаточно знать, что `protected` менее закрыт, чем `private`, но не открыт всем.

### Почему нельзя всегда делать поля `public`

Если поле открыто:

```cpp
planet.diameter = -100;
```

Объект может оказаться в неправильном состоянии. Через сеттер можно запретить неправильное значение.

### `explicit`

`explicit` запрещает неявное превращение значения в объект.

Без `explicit`:

```cpp
class Box {
public:
    Box(int size) {}
};

void printBox(Box box) {}

printBox(10); // компилятор сам создаст Box(10)
```

С `explicit`:

```cpp
class Box {
public:
    explicit Box(int size) {}
};

printBox(Box(10)); // теперь нужно писать явно
```

Для новичка правило простое: если конструктор принимает один параметр и не должен работать как преобразование типа, ставьте `explicit`.

**Исключение** — способ сообщить о нестандартной ситуации, когда функция не может нормально выполнить задачу.

Три ключевых слова:

- `throw` — бросить исключение;
- `try` — попробовать выполнить опасный код;
- `catch` — поймать и обработать исключение.

### Пример деления на ноль

```cpp
double divide(double left, double right) {
    if (right == 0) {
        throw runtime_error("Деление на ноль запрещено");
    }

    return left / right;
}

int main() {
    try {
        cout << divide(10, 0);
    } catch (const exception& error) {
        cout << "Ошибка: " << error.what() << endl;
    }
}
```

`std::runtime_error` — стандартный класс ошибки времени выполнения. Он хранит текст ошибки.

`std::exception` — общий базовый тип для стандартных исключений. У него есть метод `what()`.

### Когда исключения уместны

Исключения нужны, когда функция должна вернуть значение, но сейчас это запрещено или невозможно.

Например, геттер должен вернуть доходы компании, но по условию доступ к доходам запрещён. Возвращать `0` плохо: это выглядит как настоящие нулевые доходы. Лучше бросить исключение.

Исключения не нужны везде подряд. Для обычной проверки меню достаточно `if`.

### Геттер, который запрещает доступ

```cpp
double getIncome() const {
    if (isGovernment) {
        throw runtime_error("Доходы государственной компании скрыты");
    }

    return income;
}
```

## Ошибки компилятора простыми словами

Этот блок нужен, чтобы ученица не пугалась сообщений компилятора. Ошибка компиляции почти всегда говорит: «я не понял синтаксис» или «ты используешь то, чего здесь нет».

### `expected ';'`

**Что значит:** компилятор ожидал точку с запятой.

**Где возникает:** после объявления переменной, поля, класса, `return`, вызова функции.

Плохо:

```cpp
int age
```

Хорошо:

```cpp
int age;
```

Отдельно важно: после класса тоже нужна `;`.

```cpp
class Car {
};
```

### `was not declared in this scope`

**Что значит:** имя не объявлено в этой области видимости.

**Где возникает:** переменную не создали, забыли `#include`, ошиблись в имени.

Плохо:

```cpp
cout << age;
```

Если `age` нигде не объявлена, компилятор не знает, что это.

Хорошо:

```cpp
int age = 18;
cout << age;
```

### `private within this context`

**Что значит:** вы пытаетесь обратиться к закрытому полю или методу.

Плохо:

```cpp
car.year = 2020;
```

Если `year` находится в `private`, так нельзя.

Хорошо:

```cpp
car.setYear(2020);
```

### `no matching function for call`

**Что значит:** компилятор не нашёл функцию или конструктор с такими параметрами.

Плохо:

```cpp
Student student("Anna", 18, 10);
```

Если есть только `Student()` и `Student(string name)`, трёх параметров не существует.

Исправление: либо вызвать существующий конструктор, либо написать новый.

### `undefined reference` или `unresolved external symbol`

**Что значит:** объявление есть, а реализации нет, или `.cpp` файл не добавлен в сборку.

Часто возникает в многофайловом проекте:

```cpp
// Bed.hpp
void print() const;
```

Но в `Bed.cpp` забыли:

```cpp
void Bed::print() const {
}
```

Или компилируют только `main.cpp`, забыв `Bed.cpp`.

### `invalid operands`

**Что значит:** оператор применили к неподходящим типам.

Пример:

```cpp
cout >> age; // перепутали направление
```

Для `cout` нужен `<<`, для `cin` нужен `>>`.

### `vector subscript out of range`

**Что значит:** обращение к элементу `vector` по несуществующему индексу.

Плохо:

```cpp
vector<int> numbers = {10, 20, 30};
cout << numbers[3];
```

Индексы: `0`, `1`, `2`. Индекса `3` нет.

Исправление:

```cpp
if (index >= 0 && index < static_cast<int>(numbers.size())) {
    cout << numbers[index];
}
```
